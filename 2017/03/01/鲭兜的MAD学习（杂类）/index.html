Question-06：难以归类的各种杂项

OVA和OAD

本质上OVA和OAD其实可以算同一个东西。
OVA：Original Video Animation 原创视频动画
OAD：Original Video DVD 一般捆绑漫画进行发售

算法-CRC32

CRC：Cyclic Redundancy Check/Code 循环冗余检查/码，后面的“32”表示会产生一个32bit，即一个8位十六进制的校验值，一般由数字和字母组成。
这个校验码是用来检验下载的文件是否损坏。
文件名中有写意义不明的东西就是CRC32校验码，它是用来检验文件的完整性的，检测工具可以用WinCRC32或者其他小软件

算法-MD5

MD5：Message-Digest Algorithm 5，意为消息摘要算法第5版，它和CRC码一样也是用来检查文件的完整性的。这个算法计算的时候，所有的数据都有参与，若是被改动了一点整个值就会全然不同。MD5相当于这个文件的身份标识，偶尔会出现相同（哈希冲突），但可能性极小。

————防范传输误码的话就CRC，防范恶意破坏的一般用MD5

算法-SHA-1

同样也是哈希算法的一种，比MD5更加好，因为更加长，所以碰撞可能性更小。

BT

bittorrent。torrent是种子（本义是湍流）。P2P是Peer to Peer，P2P下载就是各拿所需，下载渠道变多了。
吸血行为：只下载不上传的用户，不提供下载渠道给他人。
ed2k：eDonkey2000，缩写eD2k，中译电驴。是国外一家商业公司开发的商业软件，后因版权问题，公司败诉，与05年倒闭。再后来一些国外业余爱好者想接着发扬光大，开发出了eMule电骡，不做商业盈利。中国出现了VeryCD，也就是中国的电驴（非授权），大部分人认为这是一个下载工具，出现了大量的吸血行为。迅雷也被称为吸血雷，因为迅雷屏蔽上传，下载过程中没有共享，下载完成之后，才会有一点点上传，而且还有限速。
基本形式ed2k://|file|<文件名称>|<文件大小>|<文件哈希值>|/
其中文件名可以随便改。比种子文件方便一点，不需要种子文件，只需要下载地址即可，但是还是要服务器的，就跟BT需要tracker服务器同理
————ed2方面推荐g国外的eMule，BT方面推荐uTorrent。
生成ed2k地址其实不难，用一个叫hashcalc的软件算出文件的ed2k哈希值，在用一个叫LinkCreator的软件直接生成文件的ed2k链接。
开源协议：GPL Gerenal Public License 这个协议要求若是使用了他人基于此协议的源代码，那么后续开发者必须也以此协议进行开源。
迅雷使用了eMule的开源代码，但是没有按照协议内容，将自己的源代码开源。迅雷基本具有了P2P所有功能但是就是不共享，不做种上传。
DHT网络：Distributed Hash Table 分布式哈希表，磁力链接 Magnet就是基于DHT网络下载的。DHT网络去中心化，不再需要服务器的支持。DHT是分布式存在的，也就是说magnet每次连接的源头是不固定的。换言之就是你查封地了BT服务器，但是你查封不了磁链。
————未来种子是会被淘汰的，主流的BT下载方式是磁链。
磁力链接的地址是magnet:?xt=urn:btih:hash值，选填没有列出来，这里是必填的形式。magnet是磁铁的意思，这里指磁链，作协议名，问号表示后面带参数，xt是exact topic 确切主题，用于在整个DHT网络寻找和定位你所要的资源，urn是Uniform Resource Name 统一资源名称，btih是BitTorrent Info Hash 种子的Hash信息，后面带hash值。
————如果找到了要下文件的hash值，只需要在前面加上magnet:?xt=urn:btih:
————或者如果找到了MD5码，那么换成magnet:?xt=urn:md5:
————或者如果找到了基于md4算法的ed2k哈希值则换成magnet:?xt=urn:ed2k:ed2kHash值
磁链的hash值一般是40位
